## ⚡ Quick Start

### Prerequisites
- [Supabase](https://supabase.com) project (free tier works)
- [Stripe](https://stripe.com) account
- [Vercel](https://vercel.com) account
- [Resend](https://resend.com) account (for emails)
- Node.js 18+ and Supabase CLI installed

### 1. Clone & Configure

```bash
# From the CorruptCLI root, run the scaffolding script:
python3 corrupt.py
```

Or manually copy `engine/` and replace all `{{TOKENS}}`.

### 2. Set Up Supabase

```bash
cd backend/supabase

# Link to your Supabase project
supabase link --project-ref YOUR_PROJECT_REF

# Push database schema
supabase db push

# Set edge function secrets
supabase secrets set \
  SUPABASE_SERVICE_ROLE_KEY=your_service_role_key \
  STRIPE_SECRET_KEY=sk_live_xxx \
  STRIPE_WEBHOOK_SECRET=whsec_xxx \
  RESEND_API_KEY=re_xxx \
  ADMIN_EMAIL=admin@yourdomain.com \
  SITE_URL=https://www.yourdomain.com

# Deploy all edge functions
supabase functions deploy --all
```

### 3. Configure Stripe Webhooks

In your Stripe Dashboard → Developers → Webhooks:
- **Endpoint URL:** `https://YOUR_SUPABASE_URL/functions/v1/stripe-webhook`
- **Events to send:**
  - `checkout.session.completed`
  - `customer.subscription.created`
  - `customer.subscription.updated`
  - `customer.subscription.deleted`
  - `invoice.payment_succeeded`
  - `invoice.payment_failed`

### 4. Deploy Frontend

```bash
cd frontend

# Install Vercel CLI if needed
npm i -g vercel

# Deploy
vercel --prod
```

### 5. Configure Admin Access

The admin portal uses OTP (one-time password) authentication. To add admin users:
1. Edit `js/admin.js` — update the `isAdmin` email check
2. Or create a `user_roles` table and check dynamically

### Security Model

```
Client Browser
    ↓ (public anon key)
Supabase (RLS enforced)
    ├── SELECT on customers/bookings → BLOCKED (anon)
    ├── INSERT/UPDATE on customers → ALLOWED (for self-service)
    ├── INSERT on bookings → ALLOWED
    └── Edge Functions (service_role key)
        └── customer-lookup → Full read access (proxy)
```

- **Row Level Security (RLS)** blocks all anonymous SELECT on sensitive tables
- **Edge Functions** serve as the only read path for customer data
- **OTP Authentication** for admin access (no passwords, no magic links)
- **Device Trust** — optional 30-day "Remember this device" for admins

## 💳 Payment Methods

The platform supports three payment paths:
1. **Pay-per-class** (Stripe Checkout) — one-time payment
2. **Class Credits** — pre-purchased credit packs
3. **Membership** — unlimited access subscriptions

Credit deduction is handled atomically by database triggers to prevent race conditions.
